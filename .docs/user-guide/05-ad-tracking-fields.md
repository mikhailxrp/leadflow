# Поля для рекламы на сайте: Яндекс.Директ и Метрика

Этот раздел — для того, **кто делает или настраивает сайт** (вебмастер, разработчик, владелец на конструкторе). Остальным его читать не нужно.

Здесь по шагам: какие два поля добавить в формы на сайте и какой маленький скрипт вставить, чтобы заявки из рекламы Яндекса правильно доходили до Лид-Канала. Написано нарочно простым языком — справится и тот, кто раньше не писал код.

---

## Зачем это нужно (на пальцах)

Представь, что каждый человек, который зашёл на сайт, приходит с **невидимой биркой на руке**. На бирке написано, откуда он пришёл: «кликнул по рекламе» или «просто зашёл».

Эта бирка нужна Яндексу, чтобы понять: **«вот этот человек кликнул по рекламе и стал клиентом»**. Тогда Яндекс показывает такую же рекламу другим похожим людям, и реклама работает лучше и дешевле.

Бирок две:

| Бирка | У кого есть | Откуда берётся |
| --- | --- | --- |
| **`yclid`** | только у тех, кто пришёл **по рекламе Яндекс.Директа** | Яндекс сам дописывает её в адрес страницы: `https://твой-сайт.ru/?yclid=ABC123...` |
| **`client_id`** | у **любого** посетителя, если на сайте стоит **Яндекс.Метрика** | это номер, который Метрика выдаёт браузеру человека |

**В чём загвоздка.** Когда человек заполняет форму и жмёт «Отправить», форма отправляет только то, что он **ввёл руками** — имя, телефон. Бирки она сама **не отправляет**. Их надо в неё положить. Ниже — как.

> 💡 Хорошая новость: как только заявки с этой формы уже доходят до Лид-Канала, ничего дополнительного в «передаче» настраивать не нужно. Две бирки просто поедут вместе с заявкой тем же путём. Твоя задача — только положить их в форму.

---

## Что нужно сделать — три шага

Одинаковы для всех сайтов. Ниже под каждым шагом есть примеры под конкретные платформы.

### Шаг 1. Добавь в форму два скрытых поля

«Скрытое поле» — это поле, которое человек **не видит**, но форма отправляет его вместе с остальными. Добавь в форму ровно два таких поля с именами `yclid` и `client_id`.

Для обычного сайта это две строчки **внутри** тега `<form>`:

```html
<input type="hidden" name="yclid" value="">
<input type="hidden" name="client_id" value="">
```

> ⚠️ **Имена менять нельзя.** Должно быть точно `yclid` и `client_id` — маленькими буквами, второе через нижнее подчёркивание. Система ищет поля ровно с этими именами. Напишешь `yClid`, `client-id` или `clientId` — не сработает.

### Шаг 2. Вставь скрипт, который заполнит эти поля

Скрипт — это короткая инструкция для браузера: «возьми бирки и впиши их в скрытые поля». Вставь этот код **один раз** в самый низ страницы, перед закрывающим тегом `</body>`:

```html
<script>
(function () {
  // 👇 ПОМЕНЯЙ это число на номер своего счётчика Яндекс.Метрики
  var COUNTER_ID = 12345678;

  // 1. Берём yclid из адреса страницы (…?yclid=ABC123)
  var yclidFromUrl = new URLSearchParams(window.location.search).get('yclid');

  // 2. Если yclid есть — запоминаем его в браузере на 90 дней,
  //    чтобы он не потерялся, пока человек ходит по другим страницам сайта
  if (yclidFromUrl) {
    document.cookie = 'lk_yclid=' + yclidFromUrl + ';path=/;max-age=7776000';
  }

  // 3. Читаем запомненный yclid (из адреса или из памяти браузера)
  function readCookie(name) {
    var found = document.cookie.match('(?:^|; )' + name + '=([^;]*)');
    return found ? decodeURIComponent(found[1]) : '';
  }
  var yclid = yclidFromUrl || readCookie('lk_yclid');

  // 4. Вписываем значение во все поля с нужным именем ИЛИ классом js-...
  function fill(name, value) {
    if (!value) return;
    document
      .querySelectorAll('input[name="' + name + '"], input.js-' + name)
      .forEach(function (input) { input.value = value; });
  }
  fill('yclid', yclid);

  // 5. Спрашиваем client_id у Метрики и вписываем в поле
  if (window.ym) {
    ym(COUNTER_ID, 'getClientID', function (clientID) {
      fill('client_id', clientID);
    });
  }
})();
</script>
```

В этом коде надо поменять **только одно** — число `12345678` на **номер своего счётчика Метрики** (тот же, что вбит в карточке «Яндекс.Метрика» на странице `Настройки → Интеграции`, поле «ID счётчика»). Больше ничего трогать не нужно.

> 💡 Скрипт умеет заполнять поле не только по имени `yclid`/`client_id`, но и по классу `js-yclid`/`js-client_id`. Это пригодится на конструкторах, где нельзя задать имя поля вручную (см. ниже).

### Шаг 3. Проверь, что заработало — без программиста

1. Открой свой сайт, дописав в конец адреса `?yclid=test123` — например `https://твой-сайт.ru/?yclid=test123`, и нажми Enter.
2. В коде формы **временно** поменяй у поля `yclid` `type="hidden"` на `type="text"` — поле станет видимым.
3. Обнови страницу. В поле должно появиться `test123` — значит, `yclid` ловится.
4. То же можно проверить со вторым полем: в нём появится длинный номер от Метрики — это и есть `client_id`.
5. Верни `type="hidden"` обратно, чтобы поля снова спрятались.
6. Отправь тестовую заявку. В карточке этого лида в Лид-Канале: `client_id` будет виден в блоке «Дополнительные поля», а `yclid` уедет в маркетинговые данные.

> 🖼️ **Скриншот — `images/05-lead-client-id.png`**
> Карточка тестового лида: видно поле `client_id` в «Дополнительных полях».

Если значения появились — всё настроено правильно.

## Как это сделать на разных платформах

### Обычный сайт (просто HTML)

Ровно то, что описано выше: две строчки `<input type="hidden">` внутри `<form>` (Шаг 1) и скрипт перед `</body>` (Шаг 2).

### Tilda

1. Открой форму → **Контент** → добавь **два скрытых поля** (тип поля «Скрытое поле» / Hidden). Задай им имена (Variable name) `yclid` и `client_id`.
2. У Tilda есть встроенная передача параметров из адреса в поля — тогда `yclid` подставится сам. Но `client_id` Метрика отдаёт только через скрипт, поэтому всё равно добавь блок **«T123 — HTML»** и вставь в него скрипт из Шага 2 (можно оставить только часть про `client_id`, если `yclid` уже подставляет Tilda).
3. Убедись, что форма отправляет заявки в Лид-Канал (вебхук Tilda уже настроен на странице `Настройки → Интеграции`).

> 💡 Имя поля в Tilda (Variable name) — это и есть то самое имя `yclid`/`client_id`, которое ищет система. Проверь, что там нет лишних пробелов и заглавных букв.

### WordPress + Contact Form 7

1. В редакторе формы добавь два скрытых поля тегами:
   ```
   [hidden yclid]
   [hidden client_id]
   ```
   Contact Form 7 сделает из них `<input name="yclid">` и `<input name="client_id">` — как раз то, что нужно.
2. Скрипт из Шага 2 вставь в подвал сайта (тема → `footer.php` перед `</body>`, либо любой плагин типа «Insert Headers and Footers»).

### WordPress + WPForms / Elementor Forms / другие конструкторы форм

В таких конструкторах имя поля напрямую задать нельзя — оно генерируется внутри. Поэтому действуем через **класс**:

1. Добавь в форму поле типа **«Скрытое поле» (Hidden Field)** — два штуки.
2. В настройках каждого поля найди **«CSS-класс»** (CSS Classes / Дополнительный класс) и впиши:
   - первому полю — `js-yclid`
   - второму полю — `js-client_id`
3. Вставь скрипт из Шага 2 в подвал сайта. Он заполнит поля по этому классу.
4. Если конструктор умеет **подставлять значение из адреса страницы** (Dynamic / Default Value → Query String), для поля `yclid` можно указать параметр `yclid` — тогда часть про `yclid` в скрипте не обязательна. `client_id` всё равно заполняется только скриптом.

> ⚠️ Важно, чтобы значение долетело до Лид-Канала **под ключом** `yclid` и `client_id`. В простых формах это имя поля. В сложных конструкторах — проверь на тестовой заявке (Шаг 3), что в карточке лида поле `client_id` действительно появилось.

### Другие конструкторы (Bitrix, Craftum, Flexbe и т.п.)

Принцип везде один:
1. Добавь **два скрытых поля** с именами `yclid` и `client_id` (или, если имя задать нельзя, — с классами `js-yclid` и `js-client_id`).
2. Вставь **скрипт** из Шага 2 в блок произвольного HTML / в подвал сайта.
3. Проверь тестовой заявкой (Шаг 3).

## Для сложных сайтов: универсальная версия

Простой скрипт из Шага 2 заполняет поля **один раз, когда страница загрузилась**. Этого хватает для обычной формы, которая уже стоит на странице. Но он **не сработает**, если форма появляется позже: всплывающее окно (попап), квиз, форма в открывающейся вкладке, или сайт-приложение (React/Vue), где страницы меняются без перезагрузки.

Чтобы работало **везде**, скрипт нужно превратить в функцию, которая дописывает метки **в момент отправки любой формы** — даже той, что появилась позже. Разберём на пальцах, что она делает:

- **запоминает** метки при загрузке страницы (`yclid` из адреса, `client_id` — как только Метрика будет готова);
- **ловит отправку любой формы** на сайте и прямо перед отправкой сама вписывает метки в скрытые поля;
- если скрытых полей в форме нет — **создаёт их сама** (не нужно добавлять поля вручную в конструкторе).

### Код функции

Вставь это в подвал сайта, перед `</body>` (там же, где стоит счётчик Метрики):

```html
<script>
/**
 * Универсальный сборщик рекламных меток yclid / client_id.
 * Вызывается один раз — в самом низу, см. следующий блок.
 */
function setupLeadTracking(counterId) {
  var store = { yclid: '', client_id: '' };

  // 1. yclid из адреса страницы → запоминаем в браузере на 90 дней
  var fromUrl = new URLSearchParams(window.location.search).get('yclid');
  if (fromUrl) setCookie('lk_yclid', fromUrl);
  store.yclid = fromUrl || getCookie('lk_yclid');

  // 2. client_id — берём у Метрики, как только счётчик готов
  store.client_id = getCookie('lk_client_id');
  whenYmReady(function (ym) {
    ym(counterId, 'getClientID', function (id) {
      if (id) { store.client_id = id; setCookie('lk_client_id', id); }
    });
  });

  // 3. На КАЖДУЮ отправку ЛЮБОЙ формы дописываем метки —
  //    даже если форма появилась на странице позже (попап, квиз, SPA).
  document.addEventListener('submit', function (event) {
    var form = event.target;
    if (!form || form.tagName !== 'FORM') return;
    ['yclid', 'client_id'].forEach(function (name) {
      if (!store[name]) return; // метки нет — не трогаем
      var input = form.querySelector('input[name="' + name + '"]');
      if (!input) { // поля нет — создаём сами
        input = document.createElement('input');
        input.type = 'hidden';
        input.name = name;
        form.appendChild(input);
      }
      input.value = store[name];
    });
  }, true); // true = срабатываем раньше отправки

  // — вспомогательные —
  function setCookie(name, value) {
    document.cookie = name + '=' + encodeURIComponent(value) + ';path=/;max-age=7776000';
  }
  function getCookie(name) {
    var m = document.cookie.match('(?:^|; )' + name + '=([^;]*)');
    return m ? decodeURIComponent(m[1]) : '';
  }
  function whenYmReady(cb) { // ждём, пока Метрика создаст window.ym
    var tries = 0;
    (function check() {
      if (window.ym) return cb(window.ym);
      if (tries++ < 40) setTimeout(check, 500); // до ~20 секунд
    })();
  }
}
</script>
```

### Где вызвать функцию

Функция выше только **описана** — сама она ничего не делает, пока её не вызвать. Вызов ставится **один раз**, ниже кода функции (можно тем же тегом `<script>`), с номером своего счётчика Метрики:

```html
<script>
  setupLeadTracking(106945257); // ← поменяй на номер своего счётчика Метрики
</script>
```

> 💡 С этой версией скрытые поля в форме можно **вообще не добавлять** — функция создаст их сама при отправке. Но если форма их уже содержит (например, ты добавил их в Tilda) — ничего страшного, функция просто впишет в них значение.

> ⚠️ Один случай, который не победит даже эта версия: конструкторы, которые отправляют форму **своим способом** и берут данные не из полей на странице, а из своей внутренней «памяти» (так делают некоторые формы на React — Elementor Pro и подобные). Для них добавь настоящее скрытое поле в самом конструкторе с именем/классом `yclid` и `client_id` — функция его найдёт и заполнит. Всегда проверяй результат тестовой заявкой (Шаг 3).

## Частые ошибки

| Ошибка | Почему не работает |
| --- | --- |
| Имя поля написали как `yClid`, `client-id`, `clientId` | Система ищет ровно `yclid` и `client_id` — маленькими буквами, второе через нижнее подчёркивание. |
| Вставили скрипт, но забыли поменять номер счётчика `12345678` | `client_id` не заполнится — Метрика не поймёт, у какого счётчика спрашивать. |
| Форма на одной странице, а `?yclid=` был на другой (человек походил по сайту) | Без запоминания в cookie `yclid` теряется. В нашем скрипте это уже учтено — не удаляй пункт 2. |
| На сайте не установлена Яндекс.Метрика | `client_id` брать неоткуда. Сначала на сайте должен стоять счётчик Метрики. |
| Скрытое поле стоит **вне** тега `<form>` | Оно не отправится вместе с заявкой. Поле должно быть внутри формы. |
| Скрипт вставили **выше** кода Метрики | `window.ym` ещё не готов. Ставь скрипт в самый низ страницы, перед `</body>`. |

## Что важно понимать

- Даже с идеально настроенными полями лид попадёт в статистику Яндекса, **только если у посетителя вообще была бирка**. Человек, который зашёл на сайт напрямую (не по рекламе) и без Метрики, — бирок не имеет. Его заявка примется как обычно, просто без рекламной метки. Это нормально.
- Эти поля **не мешают** обычной работе: заявка сохраняется в любом случае. Бирки — приятное дополнение для рекламы, а не условие приёма лида.

> ⚠️ **Про окно в 21 день (спроси у того, кто настраивает Метрику).** Метрика по умолчанию засчитывает заявку к рекламе, только если человек кликнул по ней не раньше чем 21 день назад. Это окно можно расширить, но **только вручную в самой Метрике и заранее** — задним числом оно не применяется. К полям на сайте это отношения не имеет, но знать полезно.

## Короткая памятка

1. Добавь в форму два скрытых поля: `yclid` и `client_id` (имена строго такие).
2. Вставь скрипт перед `</body>`, поменяй в нём номер счётчика Метрики.
3. Проверь через `?yclid=test123` и тестовую заявку.
4. Готово — заявки из рекламы поедут в Лид-Канал вместе с бирками.

> 📄 Технические детали (куда именно эти поля попадают в системе и какой код их читает) — для разработчиков в `.docs/modules/intake-fields-cheatsheet.md` и `.docs/modules/integrations.md`.
