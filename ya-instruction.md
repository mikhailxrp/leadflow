# Инструкция: что сделать руками (Task 1, Яндекс Директ)

Это не про код. Это про то, что можете сделать **только вы** — я не могу зарегистрировать приложение на Яндексе или получить доступ к рекламному кабинету за вас (см. `TASK.md` → «Разделение ответственности»).

Итог этих шагов: у меня будут `client_id`/`client_secret`/доступ к кабинету для research, а вы в конце примете решение **go/no-go** по фазе.

---

## Шаг 1. Зарегистрировать OAuth-приложение на Яндексе

1. Открыть https://oauth.yandex.ru/client/new
2. **Название приложения** — любое понятное, например `Лид-Канал — Яндекс Директ`.
3. **Платформы** → выбрать «Веб-сервисы».
4. **Redirect URI** — добавить (можно сразу обе строки, по одной на строку):
   ```
   http://localhost:3000/api/integrations/yandex/callback
   ```
   Продовый домен можно дописать туда же позже, когда он будет под рукой — Яндекс позволяет отредактировать список Redirect URI в настройках приложения в любой момент, без пересоздания приложения.

   ⚠️ Этого роута в коде ещё нет — он появится в Таске 2. Это нормально: сейчас просто регистрируем адрес, переходить по нему пока некуда.
5. **Доступы** → найти раздел «Яндекс.Директ» → отметить чекбокс **«Использование API Яндекс.Директа»** (это и есть scope `direct:api`). Больше ничего в этом разделе не отмечать.
6. Сохранить / создать приложение.
7. На открывшейся странице приложения скопировать и сохранить в надёжном месте:
   - **ID** — это `client_id`;
   - **Пароль** — это `client_secret`, показывается один раз. Если потеряется — на той же странице можно перевыпустить.

---

## Шаг 2. Подать заявку на доступ к API Директа

Само по себе создание OAuth-приложения доступ к API ещё не даёт — Яндекс отдельно одобряет заявку на использование API.

1. На https://yandex.ru/dev/direct/ (личный кабинет разработчика) — «Новая заявка».
2. Тип доступа — **«Тестовый»** достаточно для этого таска (работает только с песочницей, этого хватает, чтобы посмотреть реальные ответы API и макросы). «Полный» доступ (боевые кампании) можно будет запросить позже, ближе к Таскам 3–4, когда встанет вопрос о продакшене.
3. Рассмотрение занимает **до 7 дней**. Дальше по этому таску можно не ждать — но go/no-go в конце зависит от того, одобрят ли заявку.

---

## Шаг 3. Доступ к рекламному кабинету для теста

Нужен хотя бы один аккаунт Яндекс Директ, под логином которого можно будет пройти авторизацию и посмотреть, что реально приходит от API (в частности — какие поля отдаются названием, а какие ID, для таблицы макросов в спеке).

- Свой действующий рекламный кабинет — идеально.
- Если боевого кабинета нет — тестовый доступ (Шаг 2) уже даёт песочницу (Sandbox) с тестовыми данными без реального кабинета, этого достаточно для research.

---

## Шаг 4. Добавить в `.env`

В свой локальный `.env` (файл уже в `.gitignore`, наружу не уходит) добавить три переменные и заполнить первые две значениями из Шага 1:

```bash
# Яндекс Директ OAuth (Phase 22, Task 1 — research)
YANDEX_OAUTH_CLIENT_ID=
YANDEX_OAUTH_CLIENT_SECRET=
YANDEX_OAUTH_REDIRECT_URI=http://localhost:3000/api/integrations/yandex/callback
```

Ничего в `.env.example` и `CLAUDE.md` сейчас добавлять не нужно — это часть Таска 2, когда появится код, который эти переменные читает.
